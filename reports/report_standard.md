# Fixed Diet Report Standard

Цей файл є обовʼязковим стандартом показу даних для відповідей по дієті.

## Джерело даних

- Спочатку читати GitHub repo: `YuriiBeznoschenko/DIET`.
- Основний тижневий файл: `weeks/YYYY-Wxx.json`.
- Індекс ваги: `weight.csv`.
- Не відповідати по памʼяті, якщо питання про стан, вагу, їжу, тренування або журнал.
- Якщо даних немає в GitHub — прямо писати: `нема в GitHub`.
- Все неточне позначати як `estimate`.

## Основний формат відповіді

Формат не міняти без прямої команди користувача.

### 1. Головна таблиця

Колонки строго в такому порядку:

| Дата | Вага | Δ ваги | Ккал за день | Білки | Жири | Вуглеводи | Тренування | Спалено | Нетто | Звʼязок з вагою | Статус |
|---|---:|---:|---:|---:|---:|---:|---|---:|---:|---|---|

Обовʼязково показувати звʼязок з вагою: що може пояснювати зміну ваги, чи це жир, вода, тренування, сіль, вуглеводи, неповний день тощо.

### 2. Деталі їжі

Колонки строго в такому порядку:

| Дата | Прийом | Що | Кількість | Ккал | Білки | Жири | Вуглеводи |
|---|---|---|---:|---:|---:|---:|---:|

Для кожного дня обовʼязково додавати рядок:

| Дата | РАЗОМ ЗА ДЕНЬ | | | ккал | білки | жири | вуглеводи |

### 3. Тренування

Колонки строго в такому порядку:

| Дата | Тренування | Деталі | Спалено |
|---|---|---|---:|

## Правило запису в Git

Фразу `записав у Git` дозволено писати тільки після успішного commit SHA.
Після запису потрібно заново прочитати файл з GitHub і підтвердити, що запис реально є.

## Заборонено

- Не міняти форму.
- Не вивалювати сирий JSON замість таблиць.
- Не ховати БЖУ поза головною таблицею.
- Не ховати тренування поза головною таблицею.
- Не забувати рядок `РАЗОМ ЗА ДЕНЬ`.
- Не забувати колонку `Звʼязок з вагою`.
